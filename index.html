Graph Features for Insurance Fraud Detection — Belgian Commercial Insurance

Built a graph-based fraud detection model on a real-world Belgian commercial insurance dataset (6M+ nodes, 4.2M edges), engineering network features from a bipartite company–administrator graph to classify fraudulent underwriting claims with 72% accuracy and 87% sensitivity.

72%Overall model accuracy on held-out test set
87%Sensitivity — fraud cases correctly identified
6M+Nodes processed from real Belgian insurance network
4.2MEdges modelled in the company–administrator graph
Context

Standard fraud detection relies on tabular features alone. This project explored whether the structure of who manages which companies — captured as a graph — adds meaningful signal for detecting underwriting fraud in Belgian commercial insurance.

Approach

Constructed a bipartite graph in R using igraph, linking companies to administrators via 1.9M edges. Extracted degree-based network features (e.g. number of linked administrators per company) and merged them into a logistic regression model alongside company-level attributes.

Key finding

Graph-derived features were statistically significant predictors of fraud — companies with more administrator connections showed lower fraud odds, suggesting shell-company structures with fewer links are a risk signal. The model achieved 87% sensitivity, correctly flagging the majority of fraudulent cases.

Why it matters

Combines domain knowledge of commercial insurance with advanced analytical techniques rarely applied in the field — demonstrating the ability to work with large-scale real-world data and translate network theory into actionable fraud risk signals.
